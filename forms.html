Write the rationale before the score. Leave all "freedom / character" conclusions to the aggregate — no single sheet draws a verdict. Best implemented as an online table (Notion / Lark base) with the column structure below.

  3. 只能选一个平台做真实项目，你选哪个？为什么？If you could keep only one platform for real work, which, and why?

表 7 · 评分校准说明（开测前全员对齐一次）Sheet 7 · Score calibration (align once before testing)

分值Score锚点含义（以"视觉质量"为例）Anchor meaning (e.g. "visual quality")

校准流程：① 负责人选 1/3/5 分参考样片各一；② 全员先独立给这三段打分；③ 对齐理解后再开始正式测试。正式评分一律先独立、后讨论，防从众。Process: ① the lead picks one reference clip each for 1/3/5; ② everyone scores those three independently; ③ align, then start the real test. Always score independently first, discuss after — avoid herding.

命名规范（统一）Naming convention

  • 截图：平台_剧本编号_指令号_before/after，如 oppy_S07_c1_beforeScreenshots: platform_scriptID_cmd_before/after, e.g. oppy_S07_c1_before
