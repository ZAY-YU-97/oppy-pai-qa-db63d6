怎么测：选 1 部，像创作者一样深耕How to test: pick one, go deep like a creator

不追求数量——每人挑 1 个剧本，先备好参考素材，再在 References / Storyboard / Video 三个环节反复编辑、迭代，验证一件事：经过反复编辑，你最终能不能做出心里想要的影片。两个平台同样地做，填两张表，bug 发 Notion。Not about volume — each person picks one script, prepares references first, then keeps editing and iterating across References / Storyboard / Video to test one thing: after repeated editing, can you finally make the film you have in mind. Do the same on both platforms, fill two sheets, file bugs in Notion.

怎么做How it works

01

选 1 部 + 先备 ReferencePick one + prep references

从剧本库挑 1 部你想深做的。开工前，自己先收集四类参考：角色 / 场景·地点 / 风格·情绪 / 镜头。不确定长什么样？看守站案例 →Pick one from the library. Before starting, gather four kinds of references yourself: character / location / style·mood / shot. Not sure what that looks like? See the Keeper example →

02

两平台同样地做Do the same on both

同一剧本、同样的参考与措辞，分别在 Oppy 和 PAI 走一遍；编辑时两边施加完全相同的改动。Same script, same references and wording, run once on Oppy and once on PAI; apply the exact same edits to both.

03

三环节反复编辑Edit across three stages

在 References / Storyboard / Video 每个环节都主动修改、调整、重新生成、比较方案——别看到还行就进下一步。每轮留改前/改后截图。At References / Storyboard / Video, actively modify, adjust, regenerate, compare — don't move on at the first "good enough." Keep before/after shots each round.

04

填表 + 发 BugFill sheets + file bugs

两平台都做完，填① 评分表 + ② 体验表（先写依据再打分，先独立后讨论）；功能性问题按格式发 Notion，附截图/录屏。After both, fill ① score + ② experience sheets (rationale before score, independent first); functional issues to Notion in-format, with media.

开工前：先备好 ReferenceBefore you start: prepare references

真实创作者动手前都会先攒一批参考。选定剧本后，自己先按四类收集：角色（Character）· 场景/地点（Location）· 风格/情绪（Style·Mood）· 镜头（Shot）。一套满配范例见 守站案例——21 张参考图，给你看"备料备到什么程度"。Real creators gather references before they begin. After choosing your script, collect four kinds yourself: Character · Location · Style·Mood · Shot. A fully-prepared example is the Keeper case — 21 reference images showing how far to take prep.

一条底线：两边喂一样的输入与改动（语言/措辞/节奏一致），否则测的是你、不是平台。同一剧本、同一测试员、两平台都跑，才算一组对比。测试输入以中文为基准。One rule: feed identical inputs and edits to both (same language/wording/pace), or you measure the tester, not the platform. Same script, same tester, both platforms = one comparison pair. Canonical input is Chinese.

① 评分表（1–5，先写依据再打分）① Score sheet (1–5, rationale first)

A 组怎么打：别只看第一版。在三环节都主动改一改（两平台改一样的），过这几点——懂不懂意图 · 能否改并保留好的部分 · 能否精准控制 · 几轮后是否更接近目标 · 要重做几次。每项一句依据 + 分数。Scoring A: don't judge the first result. Actively edit at all three stages (same edits on both) and weigh — understands intent · edits while keeping the good parts · precise control · closer each round · how many redos. One-line rationale + score each.

开放题（两平台分别答，限 200 字内）Open questions (per platform, ≤ 200 words)

  1. 哪些地方让你感到困惑？What confused you?
  2. 哪些操作不符合直觉？What felt counter-intuitive?
  3. 哪些功能让你觉得高效、好用？What felt efficient and useful?
  4. 哪些环节让你失去控制感、或需要反复尝试？Where did you lose control or have to retry repeatedly?
  5. （差异化）在"持续编辑 → 收敛到想要的成片"这件事上，两个平台最大的差别是什么？说一个最具体的瞬间。(Differentiation) On "keep editing → converge on the cut you want," what's the biggest difference between the two platforms? Name one concrete moment.
