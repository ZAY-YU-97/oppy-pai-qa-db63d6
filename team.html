怎么测：统一、公平、可复现How to test: uniform, fair, reproducible

用同一套剧本喂两个平台、按同一套规则打分。重点测一件事——新平台主打的"编辑自由"到底兑现了没有。Feed both platforms the same scripts, score by the same rules. Above all, test one thing — whether the new platform's headline "editing freedom" actually delivered.

四条铁律（动手前先懂）Four laws (understand before you start)

1

给尺子，不给结论Rulers, not verdicts

别预设"Oppy 就是更主动/更好"。我们只记录发生了什么，结论交给汇总数据。Don't assume "Oppy is more proactive/better." Just record what happens; conclusions come from the aggregate.

2

两边喂一样的Identical inputs

语言、措辞、改法、你说话的节奏——两个平台逐字一致。语言是变量，不是平台特征。Language, wording, edits, your pacing — identical on both. Language is a variable, not a platform trait.

3

单样本不定性No verdict from one run

任何"Oppy 是 X"的判断，要跨 ≥ 5 个剧本、≥ 3 名测试员。一次不算。Any "Oppy is X" needs ≥ 5 scripts and ≥ 3 testers. Once is not enough.

4

盲评 + 先独立后讨论Blind, independent first

视觉对比隐去平台名再打分；先各自打分，再集体对齐，防从众。Hide platform identity for visual scoring; score alone first, align later — avoid herding.

测试流程The flow

准备Prep

确认账号能登录、计费面板可见；开录屏；新建空白项目；记开始时间。Confirm login & billing panel; start recording; new blank project; log start time.

执行Run

A 层创意起手 → B 层按本出片 → C 层逐条施加编辑指令。每一步两平台动作一致。Start with the layer-A seed → produce from the layer-B script → apply layer-C edits one by one. Identical moves on both.

记录Record

每个剧本测完立刻填表：客观数据、主观评分、编辑自由度、交互计数；改前/改后截图入档。Right after each script, fill the sheets: objective data, scores, editing freedom, interaction counts; file before/after screenshots.

收尾Wrap

按 平台_剧本_测试员_日期 命名导出与录屏；更新执行追踪表。Name exports/recordings platform_script_tester_date; update the tracking sheet.

顺序轮换Rotate order

A 组先 Oppy 后 PAI，B 组先 PAI 后 Oppy——抵消"先测的占便宜"。同一剧本同一测试员两平台都跑，才算一组对比。Group A does Oppy-then-PAI, Group B the reverse — cancels order bias. Only same-script, same-tester, both-platform runs form a comparison pair.

15 个剧本怎么用：三层输入Using the 15 scripts: three input layers

A

创意起点Creative seed

一句话喂进去，测 AI 从 0 生成 + 怎么跟你来回沟通。Feed the one-liner; tests zero-to-script generation + how it converses.

B

完整剧本Full script

按现成的本子出片，测执行和画面质量，也是编辑的基线。Produce from the ready script; tests execution & visuals, and is the edit baseline.

C

编辑指令Edit commands

每部 3 条，两平台逐字相同地改——这是测"编辑自由"的关键。3 per script, applied verbatim on both — the key to testing "editing freedom."

差异化怎么测How to test differentiation

H1

编辑自由 · 配对实验Editing freedom · paired

同一剧本两平台各出一版 → 施加完全相同的 C 层指令 → 逐条记「能否做到 / 方式 / 准不准 / 能否退 / 代价」+ 改前改后截图。One cut per platform from the same script → apply identical C-commands → log "done / method / precision / reversible / cost" + before-after screenshots.

H2

交互气质 · 计数Interaction · counting

行为脚本两边一致，只数事实：AI 主动提议几次、等你指令几次、总轮次、谁在推进、自我修正几次。不写"谁更主动"。Same behavior script on both, count facts only: AI proposals, times it waits, total turns, who drives, self-corrections. Don't write "who's more proactive."

这些计数是事实，可以记；"所以 Oppy 更主动"这种定性，要等多剧本多测试员汇总、并标注样本量后才能写。These counts are facts and may be logged; a verdict like "so Oppy is more proactive" comes only after aggregating many scripts/testers, with sample size stated.

Oppy 专项检查清单Oppy checklist

只测新平台。功能走通、编辑取证、边界稳定性。New platform only. Functions work, editing evidenced, beta stability.

功能走通Functions

  • 剧本生成 / Edit source 手改Script gen / manual Edit source
  • World 建角色 + 设定图World character + sheet
  • Plan 分镜 / Film 出片导出Plan board / Film export
  • @引用 / 超长本子时长处理@-refs / over-length handling

编辑取证Editing

  • 改对白 / 改角色图同步Edit dialogue / propagate art
  • 单镜定向改、是否外溢Targeted shot edit, bleed?
  • 删/加/重排镜头Delete/add/reorder shots
  • 撤销回退 / 手动接管Undo / manual take-over

边界稳定Edges

  • 生成失败 / 重试 / 丢数据Failure / retry / data loss
  • 断网恢复、是否误计费Offline recovery, mis-billing
  • 空/超长输入、矛盾指令Empty/overlong, conflicting
  • 刷新后项目是否还在Project survives refresh?

Oppy 内测会失败/重试，是已知现象——如实记录、单列，别因此整体压低体验分。Oppy's beta will fail/retry — known and expected. Record faithfully and separately; don't dock the overall score for it.

你要交什么What you hand in

  • 原始数据：每个剧本两平台的填好表单 + 录屏（按命名规范）。Raw data: filled sheets + recordings for each script on both platforms (named).
  • 对比素材：同剧本两平台成片并排剪辑；编辑自由度的改前/改后截图集。Comparison material: side-by-side cuts per script; before/after screenshot sets for editing freedom.
  • 主观汇总：评分均值 + 开放题回答；每条结论标注样本量与证据，样本不足就写"数据不足"。Subjective digest: mean scores + open answers; each conclusion notes sample size & evidence — if thin, write "insufficient data."
